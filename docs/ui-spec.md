# 打个螺丝 iOS UI Spec

本文档描述 `ScrewEverydaySpriteKit` 当前 iOS 原生 SpriteKit 版本的界面结构、布局坐标、视觉样式和交互状态，便于后续维护时快速理解页面。

## 1. 页面总览

- 应用结构：SwiftUI App Shell + `SpriteView` + 单个 `GameScene`。
- 页面数量：当前没有独立菜单、关卡选择、暂停页或设置页；主游戏、胜利页、失败页都在同一个 SpriteKit 场景内完成。
- 适配基准：逻辑画布固定为 `720 x 1280`，真实设备按等比缩放，水平居中、垂直居中。
- 方向：仅竖屏。
- 安全区：`GameView` 使用 `.ignoresSafeArea()` 和 `.statusBarHidden(true)`，游戏全屏铺满。
- 背景：AI 生成的卡通工坊背景 `app_background`，叠加模糊和深色遮罩；基础色仍保持深蓝灰 `#2D3142`。
- 当前启动页也使用同风格的 `launch_screen_art`。

## 2. 坐标与缩放规范

所有布局常量均基于左上角为原点的逻辑坐标系：

```swift
uiScale = min(sceneWidth / 720, sceneHeight / 1280)
offsetX = (sceneWidth - 720 * uiScale) / 2
offsetY = (sceneHeight - 1280 * uiScale) / 2
screenX = offsetX + logicalX * uiScale
screenY = sceneHeight - (offsetY + logicalY * uiScale)
```

设计和验收时优先使用逻辑坐标描述位置，实际屏幕由 `screenPoint(x:y:)` 转换。

## 3. 场景图层

从底到顶：

| 图层 | zPosition | 作用 |
| --- | ---: | --- |
| `boardLayer` | `1` | 木板本体、木板阴影和纹理高光 |
| `hudLayer` | `10` | 标题与实时状态文本 |
| `bufferLayer` | `15` | 备选区标签、框体、5 个空洞 |
| `boxLayer` | `20` | 顶部 4 个工具箱及箱内孔位 |
| `screwLayer` | `30` | 板上螺丝、备选区螺丝、飞行动画中的螺丝 |
| `overlayLayer` | `10000` | 胜利/失败半透明遮罩和文案 |

螺丝节点会按所属木板 `z + 100` 排序，因此上层木板的螺丝视觉上也高于下层木板。

## 4. 主游戏页

### 4.1 顶部 HUD

| 元素 | 逻辑位置 | 样式 | 文案 |
| --- | --- | --- | --- |
| 标题 | `(360, 50)` | `AvenirNext-Bold`，字号 `32`，白色 | `打个螺丝 · Day 1` |

### 4.2 顶部工具箱区

- 工具箱数量：最多 4 个。
- 单个箱体逻辑尺寸：`140 x 90`。
- 箱体间距：`16`。
- 整组水平居中。
- 中心 y：`170`。
- 每个工具箱容量：3 颗同色螺丝。
- 箱体颜色：使用目标螺丝颜色。
- 箱体结构：
  - 主体：圆角矩形，圆角 `12`，描边黑色 35% 透明度，线宽 `2`。
  - 顶部高光：白色 18% 透明度。
  - 提手：深色 `#222639` 小圆角矩形。
  - 3 个孔：沿箱体宽度均分，y 位置为箱体中心下方 `8` 逻辑点。

箱体颜色来自调色板：

| 颜色索引 | 色值 |
| ---: | --- |
| 0 | `#E63946` |
| 1 | `#F4A261` |
| 2 | `#E9C46A` |
| 3 | `#2A9D8F` |
| 4 | `#457B9D` |
| 5 | `#9B5DE5` |
| 6 | `#F15BB5` |

### 4.3 备选区

- 作用：没有同色工具箱时，螺丝临时进入备选区。
- 容量：5。
- 满 5 后再进入一颗即失败。
- 5 个孔中心 y：`310`。
- 单孔半径：`36`。
- 孔间距：`26`。
- 外框：
  - 宽度：`5 * 72 + 4 * 26 + 28 * 2 = 520`。
  - 高度：`72 + 22 * 2 = 116`。
  - 中心：`(360, 310)`。
  - 圆角：`20`。
  - 填充：`#2A2D44`。
  - 描边：`#6A6E8E`，线宽 `2`。
- 标签：
  - 位置：外框下方，逻辑 y 约 `392`。
  - 文案：`备选区（满 5 即失败）`。
  - 字体：`AvenirNext-Medium`，字号 `20`，浅灰 `white 0.72`。

### 4.4 游戏木板区

- 生成范围：逻辑 x `50...670`，逻辑 y `400...1240`。
- 基础网格：7 行 x 3 列，共 21 块木板。
- 每块木板：严格 3 颗螺丝。
- 木板尺寸从以下形状中随机选择：
  - `240 x 110`
  - `280 x 100`
  - `220 x 130`
  - `260 x 110`
  - `300 x 110`
- 木板圆角：`16`。
- 木板视觉：
  - 主体为木色圆角矩形。
  - 左下方向投影：黑色 35% 透明度，偏移 `(4, -6)`。
  - 描边：黑色 40% 透明度，线宽 `2`。
  - 顶部高光：白色 14% 透明度，高 `10`。
  - 中线木纹：黑色 18% 透明度，线宽 `1`。
- 木板颜色按 z 值循环：
  - `#A0826D`
  - `#8B6F5A`
  - `#B89576`
  - `#9C7A5E`
  - `#C4A37E`

### 4.5 螺丝

- 半径：`24`。
- 每块板 3 颗，横向分布并带轻微随机抖动。
- 可点击状态：
  - 透明度 `1.0`。
  - 外圈为螺丝颜色。
  - 白色描边 80% 透明度。
  - 黑色十字槽 85% 透明度。
  - 左上有白色高光点。
- 被遮挡状态：
  - 整体透明度约 `0.4`。
  - 阴影、描边、十字槽和高光同步变淡。
  - 不响应点击。
- 备选区中的螺丝：
  - 缩放到 `0.9`。
  - 不再响应点击。

## 5. 交互规则

### 5.1 点击螺丝

只有满足以下条件的螺丝可被点击：

- 游戏状态为 `playing`。
- 当前没有飞行动画或级联回流动画，`isBusy == false`。
- 螺丝未被更高 z 值木板遮挡。
- 螺丝仍属于某块未移除木板。

点击后：

- 若存在同色且未满的工具箱，螺丝飞入工具箱下一个孔位。
- 若不存在同色工具箱，螺丝飞入备选区下一个空位。
- 若备选区已满且还要进入，触发失败页。

### 5.2 动画

- 入箱动画：移动到箱孔、缩放到 `0.55`、旋转 `1.5π`，时长 `0.30s`。
- 入备选区动画：移动到备选孔、缩放到 `0.9`、旋转 `π`，时长 `0.34s`。
- 备选区回流入箱：移动到箱孔、缩放到 `0.55`、旋转 `1.5π`，时长 `0.32s`。
- 备选区重排：剩余螺丝左对齐移动，时长 `0.22s`。
- 工具箱满 3 后闪光：白色闪层淡出并放大到 `1.15`，时长 `0.26s`。
- 首屏入场：木板和螺丝淡入，木板从下方 `40` 逻辑点上移复位。

动画期间必须锁定输入，防止连续点击破坏状态。

### 5.3 工具箱补箱与备选回流

- 工具箱收满 3 颗后，从颜色池补充新颜色。
- 如果颜色池为空，该工具箱永久移除。
- 每次补箱或移除工具箱后，自动扫描备选区：
  - 找到第一个能匹配任意工具箱颜色的螺丝。
  - 该螺丝飞入对应工具箱。
  - 备选区剩余螺丝左对齐。
  - 如果工具箱再次满 3，继续触发补箱和回流。

## 6. 胜利页与失败页

胜利和失败不是独立 SwiftUI 页面，而是 `overlayLayer` 覆盖在当前游戏场景之上。

### 6.1 通用覆盖层

- 遮罩：全屏矩形。
- 填充：黑色 72% 透明度。
- zPosition：`overlayLayer` 内底层。
- 点击行为：任意位置点击都会重新随机生成一关。

### 6.2 胜利页

| 元素 | 位置 | 样式 | 文案 |
| --- | --- | --- | --- |
| 标题 | 屏幕中心上方 `60` 逻辑点 | `AvenirNext-Bold`，字号 `72`，绿色 `#2A9D8F` | `通关!` |
| 提示 | 屏幕中心下方 `40` 逻辑点 | `AvenirNext-Medium`，字号 `28`，白色 | `点击屏幕重开（新关卡）` |

触发条件：

- 所有工具箱都被移除。
- 已处理螺丝数达到总数。
- 备选区为空。
- 游戏仍处于 `playing`。

### 6.3 失败页

| 元素 | 位置 | 样式 | 文案 |
| --- | --- | --- | --- |
| 标题 | 屏幕中心上方 `60` 逻辑点 | `AvenirNext-Bold`，字号 `72`，红色 `#E63946` | `失败` |
| 提示 | 屏幕中心下方 `40` 逻辑点 | `AvenirNext-Medium`，字号 `28`，白色 | `点击屏幕重开（新关卡）` |

触发条件：

- 备选区已有 5 颗螺丝。
- 玩家又点击了一颗无法进入任何工具箱、只能进入备选区的螺丝。

## 7. 下一版页面设计方案

以下页面为建议新增内容，当前版本尚未实现。目标是把单局玩法扩展成“可连续闯关、可追目标、可收集”的轻量关卡制体验。

### 7.1 首页

首页定位：玩家进入游戏后的主入口，承担主题故事、继续游戏、奖励目标和收藏入口。

主题故事建议：

```text
螺丝小镇的机器塔每天都会松动。
玩家是新来的修理师，需要把散落在木板机关里的彩色螺丝拧回工具箱，
逐层修复机器塔，解锁小镇里的工坊、齿轮钟、飞艇和秘密零件。
```

故事不需要长篇展示，首页只保留一句短文案，更多内容通过关卡章节和图鉴逐步展开。

#### 首页布局 (基于 asset_all_fal 效果图)

首页采用像素级定位的叠加图层布局，基准设计稿尺寸为 `750 x 1342`。

| 图层 ID | 作用 | 逻辑尺寸 (W x H) | 相对坐标 (X, Y) | 适配 zIndex | 说明 / 交互绑定 |
| :--- | :--- | :---: | :---: | :---: | :--- |
| `stamina_bar` | 修理值/体力 | 284x99 | (0, 0) | 10 | 叠加显示当前修理度，文本水平偏移 +24 避开图标 |
| `coin_bar` | 金币显示 | 257x99 | (493, 0) | 11 | 叠加显示当前金币数，文本水平偏移 +24 避开图标 |
| `daily_btn` | 每日任务按钮 | 130x191 | (6, 475) | 12 | 绑定每日任务弹窗，左侧排列 |
| `rank_btn` | 排行榜按钮 | 130x191 | (6, 648) | 13 | 绑定排行榜展示，左侧排列 |
| `pets_btn` | 收藏图鉴按钮 | 130x185 | (6, 814) | 14 | 绑定收藏图鉴页面，左侧排列 |
| `shop_btn` | 关卡列表按钮 | 133x210 | (617, 654) | 15 | 绑定关卡选择页，右侧排列 |
| `settings_btn` | 设置按钮 | 133x197 | (617, 839) | 16 | 弹出设置半屏抽屉，右侧排列 |
| `play_btn` | 开始修理按钮 | 407x222 | (173, 1086) | 17 | 主按钮，带有 1.03 缩放的微弱呼吸动画 |

#### 布局位置与间距规格

1. **顶部资源栏间距**：
   - `stamina_bar` (左侧) 与 `coin_bar` (右侧) 之间水平净间距为 `493 - 284 = 209` 逻辑点。
   - 资源条贴顶展示，与状态栏或屏幕边缘对齐。

2. **左侧按钮组分布**：
   - 三个按钮（`daily_btn`、`rank_btn`、`pets_btn`）左对齐，X 坐标固定为 `6` 逻辑点。
   - 垂直间距：
     - `daily_btn` 到 `rank_btn` 的垂直偏移距离为 `648 - 475 = 173` 逻辑点。
     - `rank_btn` 到 `pets_btn` 的垂直偏移距离为 `814 - 648 = 166` 逻辑点。

3. **右侧按钮组分布**：
   - 两个按钮（`shop_btn`、`settings_btn`）右对齐，X 坐标固定为 `617` 逻辑点。
   - 垂直间距：
     - `shop_btn` 到 `settings_btn` 的垂直偏移距离为 `839 - 654 = 185` 逻辑点。

4. **底部主按钮定位**：
   - `play_btn` (开始游戏) 尺寸较大，位于底部区域。
   - X 坐标为 `173` 逻辑点，右侧间距为 `750 - (173 + 407) = 170` 逻辑点，基本实现水平居中。
   - Y 坐标为 `1086` 逻辑点，保留底部安全区域。

#### 首页视觉

- 背景色与游戏主场景一致，使用 `mockup_cute_clay` 大图作为背景，延续粘土/卡通美术风格。
- 交互控件素材全部来自 `asset_all_fal.png` 效果图切图，通过精确的 Alpha-Masked NCC 匹配算法实现像素级去背（去纯白色 #FFFFFF）并对齐，避免重叠处的穿帮。
- 为主按钮 `play_btn_fal` 添加柔和的持续呼吸动效，周期为 `0.95s`，提振点击反馈。
- 数值叠加使用 `AvenirNext-Bold` 风格或系统的 `.rounded` 粗体字，字号为 `20` 乘以缩放系数，前景色为纯白色。

#### 首页交互

- 点击主按钮：
  - 有未完成关卡时进入当前关卡。
  - 没有进度时进入第 1 关。
- 点击关卡：进入关卡页面。
- 点击图鉴：进入收藏图鉴页面。
- 点击奖励：进入奖励进度弹层或结算奖励说明页。
- 首页可播放轻微机关待机动画，但不能影响按钮响应。

### 7.2 关卡页面

关卡页定位：让玩家知道自己在推进章节，并能快速继续下一关。

#### 关卡结构

- 推荐采用章节制，而不是一次展示无限列表。
- 每章 20 关。
- 每 5 关一个小节点奖励。
- 每 20 关一个章节奖励和主题升级。

章节示例：

| 章节 | 名称 | 主题 |
| ---: | --- | --- |
| 1 | 松动的工坊 | 基础木板、普通螺丝 |
| 2 | 齿轮钟楼 | 更密集遮挡、更多颜色 |
| 3 | 蒸汽飞艇 | 限时奖励、特殊木板外观 |
| 4 | 秘密零件库 | 更复杂补箱顺序 |

#### 关卡页布局

| 区域 | 逻辑位置/尺寸 | 内容 | 说明 |
| --- | --- | --- | --- |
| 顶栏 | y `36...110` | 返回、章节名、奖励箱入口 | 返回首页 |
| 章节进度 | x `70...650`，y `130...220` | `松动的工坊 12/20`、横向进度条 | 展示当前章节推进 |
| 关卡地图 | x `70...650`，y `250...1040` | 弯曲路径节点或 4 列网格节点 | 第一版建议 4 列网格，开发更稳 |
| 当前关卡卡片 | x `70...650`，y `1060...1190` | 当前关卡、难度、奖励、开始按钮 | 固定底部操作区 |

#### 关卡节点状态

| 状态 | 视觉 | 行为 |
| --- | --- | --- |
| 已通关 | 彩色螺丝圆章、显示星标或勾 | 可重玩 |
| 当前关 | 发光描边、轻微缩放呼吸 | 点击开始 |
| 未解锁 | 灰色木牌、锁图标 | 点击提示“先完成上一关” |
| 奖励关 | 宝箱或扳手图标 | 达成后弹出奖励 |

#### 关卡页继续驱动

- 当前关卡卡片始终展示“下一关可获得什么”。
- 每 5 关显示一个明确奖励节点，例如：
  - `第 5 关：金币 +50`
  - `第 10 关：稀有螺丝图鉴`
  - `第 15 关：工具箱皮肤碎片`
  - `第 20 关：章节宝箱`
- 如果玩家失败，返回关卡页时当前关节点保持高亮，不推进进度。

### 7.3 结算页面

结算页定位：完成一局后展示结果、奖励、下一目标，并推动继续下一关。

建议拆成胜利结算和失败结算两种状态，但共用同一套弹层结构。

#### 胜利结算布局

| 区域 | 逻辑位置/尺寸 | 内容 | 说明 |
| --- | --- | --- | --- |
| 半透明遮罩 | 全屏 | 黑色 72% 透明 | 延续当前结果页 |
| 结果标题 | y `230...310` | `修理完成!` | 替代当前 `通关!`，更贴合主题 |
| 关卡摘要 | y `330...390` | `第 12 关 · 用时 01:36` | 可追加剩余备选格、连消次数 |
| 奖励卡 | x `70...650`，y `430...610` | 金币、经验、图鉴碎片 | 主奖励展示 |
| 奖励进度 | x `70...650`，y `650...780` | `再玩 2 分钟领取扳手宝箱`、进度条 | 关键留存目标 |
| 下一奖励预告 | x `90...630`，y `800...900` | `第 15 关解锁：彩虹螺丝图鉴` | 引导继续闯关 |
| 按钮区 | y `980...1130` | `下一关`、`回首页`、`重玩` | `下一关` 为主按钮 |

#### 失败结算布局

| 区域 | 内容 |
| --- | --- |
| 结果标题 | `备选区满了` |
| 原因提示 | `没有同色工具箱时，螺丝会进入备选区。备选区满 5 格后会失败。` |
| 保底奖励 | 少量金币或今日进度，不给主要关卡奖励 |
| 进度条 | 展示“再尝试 1 次领取鼓励奖励”或保留今日游玩进度 |
| 按钮 | `再试一次` 为主按钮，`回首页` 为次按钮 |

失败页不建议只显示“失败”，需要给出具体失败原因，帮助玩家理解规则并愿意再试。

#### 奖励类型

| 奖励 | 用途 | 是否第一版需要 |
| --- | --- | --- |
| 金币 | 通用软货币，可用于提示/皮肤 | 可选 |

#### 奖励进度条

结算页建议至少保留一个“短周期目标”：

```text
今日修理进度 3/5
[██████░░░░] 再通关 2 关领取扳手宝箱
```

也可以使用游玩时长目标：

```text
今日专注修理 06:20 / 10:00
[██████░░░░] 玩满 10 分钟领取齿轮宝箱
```

第一版建议优先使用“通关数进度”，不要用真实时长作为核心奖励条件。通关数和玩法表现更相关，也更容易被玩家理解。

### 7.4 收藏图鉴页面

建议需要收藏图鉴，但第一版应做成轻量长期目标，而不是复杂养成系统。

是否需要的判断：

- 如果目标只是做一个单局小游戏，可以不做图鉴。
- 如果希望玩家继续完成更多关卡，建议做图鉴，因为它能提供“下一关之外”的长期收集理由。
- 图鉴不应该影响主玩法胜负，不要让玩家为了数值被迫收集。

#### 图鉴主题

推荐收藏对象为“修理零件”，比单纯收藏颜色螺丝更有故事感：

| 图鉴类型 | 示例 | 解锁方式 |
| --- | --- | --- |
| 特殊螺丝 | 彩虹螺丝、星星螺丝、蒸汽螺丝 | 关卡奖励、章节宝箱 |
| 工具箱 | 旧木箱、红铁箱、蒸汽箱 | 收集碎片解锁 |
| 机器部件 | 小齿轮、压力表、铜管、飞艇桨叶 | 每章主线奖励 |
| 成就徽章 | 3 连清箱、无备选通关、快速通关 | 达成玩法成就 |

#### 图鉴页布局

| 区域 | 逻辑位置/尺寸 | 内容 |
| --- | --- | --- |
| 顶栏 | y `36...110` | 返回、标题 `零件图鉴`、收集进度 |
| 分类标签 | y `130...200` | `螺丝`、`工具箱`、`机器部件`、`徽章` |
| 图鉴网格 | x `50...670`，y `230...1080` | 3 列卡片网格 |
| 详情区/弹层 | 居中弹层 | 名称、稀有度、来源、故事短句 |

#### 图鉴卡片状态

| 状态 | 视觉 | 文案 |
| --- | --- | --- |
| 未获得 | AI 零件图暗色剪影、问号覆盖 | `第 10 关后可获得` |
| 碎片中 | 半透明图标、碎片数 | `2/5` |
| 已获得 | 彩色 AI 零件图标、稀有度描边 | 名称和短描述 |
| 新获得 | 角标 `NEW`、轻微闪光 | 首次进入时播放 |

#### 图鉴对继续闯关的作用

- 关卡页显示“下一关可能获得的图鉴碎片”。
- 结算页展示本局获得的碎片和距离解锁还差多少。
- 首页今日目标可展示“再通关 2 关，获得零件宝箱”。
- 图鉴详情里的来源要能反向跳转到对应章节或关卡。

### 7.5 推荐最小版本

如果要控制开发量，建议按以下顺序实现：

1. 首页：主按钮、关卡入口、今日通关进度。
2. 关卡页：章节进度、关卡节点、开始当前关。
3. 结算页：胜利/失败原因、奖励列表、下一关按钮。
4. 轻量图鉴：只做 1 个分类“机器部件”，每章解锁 1 个。
5. 后续再扩展金币、皮肤、道具和多分类图鉴。

第一版不要同时引入太多经济系统。最重要的是让玩家清楚知道：“我现在在哪一关，下一关能拿什么，继续玩能修好什么东西。”
