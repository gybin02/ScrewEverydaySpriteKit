# 外围系统界面设计规范 (app_ui_screens)

本文档定义了游戏主菜单（首页）、关卡选择、图鉴手账以及通关结算界面的 UI 设计规格，同时包含了首页素材的自动定位坐标信息。

---

## 1. 主界面 (首页) 规范

首页采用像素级定位的叠加图层布局，基准设计分辨率为 **608 × 1280**。

### 1.1 视觉风格
* **背景底图**：使用 [bg_cute_clay.png](file:///Users/yd-sz-dn0588/Downloads/game/ScrewEverydaySpriteKit/art/spec/bg_cute_clay.png)，维持统一的 3D 卡通黏土风格。
* **交互控件**：全部来自 [art/spec/](file:///Users/yd-sz-dn0588/Downloads/game/ScrewEverydaySpriteKit/art/spec/) 目录下的高精度去白边素材。
* **主按钮特效**：为 `play_btn` (开始修理) 按钮添加柔和的持续呼吸动效，周期为 `0.95s`，缩放范围 `1.0 ~ 1.03`，以提振玩家的点击欲望。
* **数值字体**：使用 `AvenirNext-Bold` 风格或系统的 `.rounded` 粗体字，字号为 `20` 乘以缩放比率，前景色为纯白色。

### 1.2 绝对定位与间距规格
* **顶部资源栏间距**：体力条右侧与金币条左侧水平间距为 `428 - (18 + 178) = 232px`。
* **左侧按钮组垂直分布**：
  * 三个按钮（`daily_btn`、`rank_btn`、`pets_btn`）左对齐悬浮。X 坐标固定为 `16px`。
  * `daily_btn` 到 `rank_btn` 间距为 `13px`；`rank_btn` 到 `pets_btn` 间距为 `16px`。
* **右侧按钮组垂直分布**：
  * 两个按钮（`shop_btn`、`settings_btn`）右对齐悬浮。X 坐标固定为 `505px`。
  * `shop_btn` 到 `settings_btn` 垂直间距为 `12px`。
* **底部主按钮定位**：
  * `play_btn` (开始游戏) 绝对 X 轴为 `142px`，右侧距边 `142px`实现完美水平居中；绝对 Y 轴为 `1048px`，距底边 `83px`。

## 2. 关卡选择界面 (Levels Screen)

关卡页定位为章节闯关系统，帮助玩家快速浏览和推关。

* **章节制设计**：每章 20 关，每 5 关设置一个小节点奖励，每 20 关设置一个章节总大奖。
* **章节示例**：
  * 第一章：`新手车间` (关卡 1-20，掉落：基础螺丝刀、垫片、螺母)
  * 第二章：`时钟工坊` (关卡 21-40，掉落：黄铜齿轮、发条轴承、钟摆摆轮)
  * 第三章：`飞艇码头` (关卡 41-60，掉落：气囊蒙皮、推进螺旋桨、高度计)
* **节点UI样式**：
  * 普通关卡节点：小圆点，中间显示关卡号。
  * 特殊奖励节点（每 5 关）：带有宝箱/螺丝刀/螺母小图标的稍大节点。
  * 路线表现：关卡间使用虚线/实线相连，构成线性的地图轨迹。

## 3. 图鉴手账界面 (Illustrated Handbook)

用于展示玩家通关后收集到的所有零件和机器拼装进度。

* **分层标签页 (Tabs)**：
  * `全部部件`：展示当前已经获得的稀有零件。
  * `我的工坊`：展示零件拼装成完整机器（如时钟、飞艇）的进度与草图。
* **收集节点数据结构**：
  * 每当通关掉落 `collectionName` 的关卡时，将本地存档中的 `IllustrationItem` 标记为 `unlocked = true`。
  * 在图鉴页中，已解锁的零件以全彩卡片高亮显示，附带名称与解锁日期；未解锁的零件显示灰色占位，并提示其掉落的来源关卡（如 `掉落来源：第 25 关`）。

## 4. 通关结算界面 (Settlement Screen)

单局游戏胜利/失败后，退出 SpriteKit 并弹出 SwiftUI 结算页：

* **胜利结算展示**：
  * 信息概览：通关时长、拧下的螺丝总数、移动步数。
  * 通关奖励：常规金币值（如 `+10 金币`）以及图鉴零件解锁提示。
  * 行动按钮：`继续下一关` (Primary，触发下一关) 以及 `返回关卡列表` (Secondary)。
* **失败结算展示**：
  * 失败原因分析：例如 `备选区满员！`。
  * 动作按钮：`重新挑战` (Primary) 以及 `返回首页`。
