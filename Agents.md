# Agents 开发记录

本文档记录 AI 辅助开发过程中的关键技术决策和经验总结。

---

## 关键技术决策

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| 工程管理 | XcodeGen + `project.yml` | 避免 `.xcodeproj` 冲突，文件变更自动同步 |
| 图片加载 | `Resources/` 目录 + `UIImage.bundled()` | 不用 imageset 拖拽流程，适合 AI 批量生成 |
| 本地化 | String Catalog（`.xcstrings`） | Xcode 原生方案，编译期校验，英语为主中文适配 |
| 存档 | UserDefaults | MVP 阶段数据量小，足够简单 |
| 坐标系 | 逻辑画布 720×1280 等比缩放 | 与 JS 原版一致，方便对照移植 |
| 动画锁 | `isBusy` 全局锁 | 防止飞行动画期间连续点击破坏状态 |

---

## AI 辅助开发经验

### 有效实践

1. **先规则后代码**：在 README 中完整描述游戏规则，AI 根据规则生成实现代码，减少歧义。
2. **分离关注点**：数据模型与视觉节点分离，AI 修改逻辑时不容易破坏渲染。
3. **固定 seed 验证**：用固定 seed 保证关卡可复现，截图对比发现问题。
4. **渐进式迭代**：每个阶段独立验收（编译通过 + 模拟器运行 + 截图确认），再进入下一阶段。
5. **文档驱动**：用 spec 文档描述目标状态，AI 对照 spec 实现，减少来回沟通。

### 注意事项

1. SpriteKit 的 zPosition 和触摸检测需要手动管理，AI 容易遗漏层级冲突。
2. SwiftUI + SpriteKit 混合架构中，状态同步需通过回调闭包，不能直接绑定 `@State`。
3. XcodeGen 的 `resources` 配置必须显式声明，否则图片/音频不会被打包。
