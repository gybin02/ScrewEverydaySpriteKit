# 切图与导出规范

当任务包含“切图”“图片转代码 + 资源”“导出 PNG”“图标单独导出”“当前选中的 UI 图片还原”时，必须使用本规范。

## 顺序要求

必须先创建 `layers.manifest.json`，再导出切图和写代码。manifest 至少包含每个资源的 `source_bbox`、缩放后 bbox、类型、z-index 和输出路径。

禁止先写一个近似页面，再临时找素材补图。这样会导致布局、切图、透明背景全部跑偏。

切图前必须生成 bbox 预览图：

```bash
scripts/preview_bboxes.py source.png layers.manifest.json qa/bbox-preview.png --only-type bitmap
```

检查红框是否完整覆盖每个切图元素。红框不准时，先改 manifest，不能直接切图。

## 资源分类

按类型分别导出到不同目录：

- `assets/icons/`：按钮图标、导航图标、功能图标、状态图标
- `assets/images/`：头像、产品图、截图、照片、背景图
- `assets/illustrations/`：复杂插画、不规则装饰图
- `assets/charts/`：复杂图表或数据可视化整体图层

使用稳定命名：

```text
icon-<区域>-<用途>-<序号>.png
image-<区域>-<用途>-<序号>.png
illustration-<区域>-<用途>-<序号>.png
chart-<区域>-<用途>-<序号>.png
```

示例：

```text
icon-tab-home-01.png
image-card-product-03.png
illustration-hero-bg-01.png
chart-dashboard-sales-01.png
```

## PNG 强制规则

- 每个图标和位图都必须导出为独立透明背景 PNG。
- 每个 PNG 必须来自当前源图中的对应元素，不得使用相似素材、图标库、生成图或手工近似重绘替代。
- PNG 画布必须等于缩放后的元素外框尺寸。
- 必须保留元素外框内的透明留白。
- 禁止裁掉透明像素。
- 禁止按可见内容自动裁切。
- 禁止内容自适应裁边、智能缩边、shrink-to-fit、auto bounds。
- 除了应用统一的 750px 全局缩放比例，不得二次缩放资源。
- 禁止把多个图标或多个位图合并成一个文件。
- 禁止把图标或位图转换为矢量路径。
- 需要透明背景的 PNG 必须保留 alpha 通道，背景 alpha 必须为 0。
- 白底、灰底、页面底色、卡片底色被带进 PNG 时，判定切图失败，除非该底色是元素视觉本身。
- 推荐使用 `scripts/extract_png_asset.py` 或等效流程按 bbox 从源图导出；该流程必须保证不 trim、不改画布尺寸。
- 对需要透明背景的切图，必须运行 `scripts/audit_png_assets.py --require-transparent-bg` 或等效检查；白色/灰色矩形底、无 alpha 通道、角落不透明都判定失败。

## 当前源图唯一性

切图资源只能来自本次任务的源图或用户明确提供的原始素材。不得复用其他文件夹、其他项目、历史会话、在线素材库或组件库中的视觉资产。

- 头像必须从原图头像位置提取，不得换成其他头像。
- 状态栏图标必须按原图形态还原；无法精确代码化时提取 PNG。
- 会员皇冠、底部导航、服务入口、礼物、钱包、徽章、星星、勾选章等都必须来自原图对应区域。
- 卡片里的插画必须保持原图的大小、位置、角度、露出比例和裁切方式。
- 如果提取透明 PNG 困难，允许先导出完整元素外框裁片并继续抠背景，但不允许换成相似插画。

## 防止只切到局部

切图失败最常见原因是按抠除后的可见主体重新计算边界，导致图标只剩一段、圆形底被裁、阴影被削或半透明边缘丢失。必须按下面规则处理：

- 先确定原图中的完整元素外框，再抠背景；不得先抠图再用抠图结果重新裁边。
- 完整元素外框必须通过 bbox 预览图确认；不能只靠肉眼估坐标后直接导出。
- 元素外框包括深色图标主体、浅色底形、阴影、光晕、模糊、抗锯齿和必要透明留白。
- 如果图标在浅色圆形、八边形、胶囊形底上，必须把该底形作为完整视觉范围处理；若底形用 CSS 复现，则 PNG 图标主体仍必须保留足够透明留白。
- 如果边界不确定，向四周扩展 `4px-12px` 安全留白；禁止缩小。切图宁可透明画布稍大，也不能缺失或贴边。
- 导出后检查 alpha 通道：只要非透明像素触碰 PNG 的上、下、左、右任意边缘，就默认判定为裁切风险，必须扩大画布重出。
- 半透明像素也算内容，不能因为透明度低就裁掉。
- 对服务宫格、功能入口、底部导航图标，必须整件导出：不能只截耳机的一侧、礼盒的一角、徽章的一段或图标内部符号。
- 对卡片插画，必须按原图外框和露出范围导出：不能把钱包、礼盒、徽章、星星重新居中、放大、缩小或换角度。
- 不允许用自动 trim、自动 bbox、透明像素裁剪后的尺寸作为最终切图尺寸。

## 背景抠除规则

当图标或位图嵌在白色、灰色或纯色背景上，并且该背景不是设计内容本身时：

1. 只移除背景像素。
2. 保持原元素画布尺寸不变。
3. 保留抗锯齿边缘和半透明像素。
4. 在浅色、深色和棋盘格背景上检查边缘。
5. 如果出现白边、灰边、缺角、阴影被裁切或边缘丢失，必须重出。

正确顺序：

1. 按完整元素外框向四周外扩安全留白。
2. 从源图裁出固定画布 PNG。
3. 在该固定画布内抠除背景。
4. 保存 alpha PNG。
5. 禁止 trim 或按主体重新裁切。

如果白色、灰色或纯色区域本身就是设计的一部分，必须保留。

透明背景验收：

- 在棋盘格背景上预览，不能出现白色/灰色矩形块。
- 在黑色背景上预览，不能出现浅色脏边。
- 在白色背景上预览，不能丢失浅色主体或半透明阴影。
- 检查 PNG 模式必须包含 alpha；没有 alpha 通道的 PNG 不能作为透明切图交付。
- 如果预览时出现白色或灰色方块背景，直接判定失败。

## 复杂内容规则

复杂图表、数据可视化、手绘图形、不规则插画或无法稳定拆成简单形状的视觉组：

- 整体保留为透明 PNG 图层。
- PNG 画布等于元素外框。
- 图中的可读文字尽量单独还原为可编辑文本，覆盖在 PNG 上方对应位置。
- 禁止强行拆成手写 SVG 路径。

## 分模块验收

必须分三类独立验收，不能混检：

1. **矢量图层模块**

   检查 x/y 坐标、宽高、圆角、描边、颜色、渐变、阴影、透明度和层级顺序。

2. **文本图层模块**

   检查文字内容、x/y 坐标、宽高、字体、字号、字重、行高、字间距、颜色、对齐方式和换行。

3. **位图/图标切图模块**

   逐个检查 PNG 的宽度和高度，分别与 750px 缩放后的对应元素外框比对。把 PNG 叠放到最终坐标位置，确认没有边缘缺漏、透明留白丢失、阴影裁切、白边、灰边或异常背景。

   额外检查 alpha 边缘：PNG 四边不得有非透明像素贴边。贴边通常表示切小了，必须扩大画布重出。

   额外检查来源一致性：PNG 叠到原图对应位置时，形状、颜色、比例、角度、裁切和风格必须一致。只要看起来像另一套素材，就判定失败。

任一类别出现裁切问题时，该类别必须全部重出图，再重新验收。

## 整体画板复核

矢量、文本、位图/图标全部拼装完成后：

- 截取最终 750px 画板截图。
- 与 750px 原图底图叠加比对。
- 确认无缺图、无裁切、无偏移、无层级错误、无非预期布局变化。
- 如果仍有差异，必须说明原因，例如原字体缺失、原图像素不足、元素被遮挡或源图本身压缩严重。
